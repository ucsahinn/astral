# Astral v2.2.13 - Scoped AllowedApps hotfix

Tarih: 2026-06-18

v2.2.13, v2.2.12 ustune gelen dar kapsam hotfix'idir. Amac, Astral bagli gorunurken secili hedefler disindaki normal internetin etkilenmemesini daha sert dogrulamak ve WireSock uygulama kapsam satirini dogrudan `AllowedApps` olarak yazmaktir.

## Degisenler

- WireSock profili artik plain `AllowedApps = ...` direktifiyle uretilir.
- Eski veya genis `AllowedApps` ve `#@ws:AllowedApps` satirlari profil yeniden yazilirken temizlenmeye devam eder.
- Web hedeflerinde genel tarayici exe'leri profile eklenmez; yalniz `Astral.WebProxy.exe` kapsama girer.
- Live smoke testi secili olmayan `www.microsoft.com:443` baglantisinin Astral bagliyken DIRECT kalabildigini kritik kontrol olarak dogrular.
- Live smoke testi profilin plain `AllowedApps` kullandigini ayrica raporlar.

## Kullanici Etkisi

- Secili hedefler disindaki trafik normal baglantida kalmalidir.
- Astral, tum PC veya tum tarayici VPN'i gibi davranmayacak sekilde profil kapsam sinirini daha acik yazar.
- Tanimlama paketleri ve smoke ciktisi, uygulama kapsami ve secili olmayan internet sagligi icin daha net kanit verir.

## Guvenlik Notu

Web hedefleri HTTPS icerigini cozmez; CONNECT host allowlist'iyle calisir. Genel tarayici processleri WireSock `AllowedApps` kapsaminda degildir. Paket imzasi yoksa guncelleme guven siniri GitHub release yolu, asset digest, SHA-256 dosyasi ve manifest eslesmesidir.

## Kalan Riskler

- Canli servis erisimi ISP, servis tarafi degisiklikleri ve yerel guvenlik yazilimlarina gore degisebilir.
- Kurumsal PAC/proxy veya tarayici DoH/QUIC ayarlari web hedeflerinde ek dogrulama gerektirebilir.
- Kod imzalama sertifikasi yapilandirilmadiysa Windows SmartScreen uyarisi gorulebilir.
