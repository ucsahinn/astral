# Discorder v2.1.3 - Tarayıcı modu ve uygulama içi sürüm notları

Yayın tarihi: 2026-06-15

v2.1.3, v2.1.2 sonrası gelen tanılama kayıtlarından çıkan tarayıcı modu sorununu ve küçük ürün akışı eksiklerini hedefleyen patch sürümdür. Ana davranış değişmedi: Discorder Discord bağlantısını yönetir; tarayıcı kapsamı yalnızca kullanıcı Tarayıcı modu açarsa dahil edilir.

## Kısa Özet

- Tarayıcı modu açıkken Chrome, Brave ve Edge yolları WireSock profiline güvenli biçimde yazılır.
- Alt aksiyon çubuğuna uygulama içi sürüm notları butonu eklendi.
- Alt aksiyon çubuğuna Discorder'ı yeniden başlatan buton eklendi.
- Firewall / hosts temizleme scriptleri yoğun makinelerde daha uzun süre bekleyebilir.
- Progress, hata, onarım ve tarayıcı modu metinleri daha net hale getirildi.

## Neden Önemli?

Tanılama kayıtlarında Tarayıcı modu açıkken desteklenen tarayıcıların profile dahil edildiği görülüyordu; buna rağmen tarayıcıdaki `discord.com` akışı bazı makinelerde beklenen şekilde çalışmıyordu. Chrome, Brave ve Edge gibi uygulamaların tam dosya yolları `C:\Program Files` altında boşluk içerir. Bu sürüm, `AllowedApps` satırındaki boşluklu yolları tırnaklayarak WireSock tarafında yanlış ayrıştırılma riskini kaldırır.

Kapanış sırasında bazı makinelerde firewall temizliği 20 saniyelik PowerShell sınırına takılabiliyordu. v2.1.3 bu sınırı 60 saniyeye çıkarır; işlem hâlâ başarısız olursa tanılama içinde görünür kalır.

## Değişiklikler

- `AllowedApps` üretimi boşluk içeren tam uygulama yollarını tırnaklar.
- Güvensiz profile enjeksiyonu engellemek için uygulama kapsamı değerlerinde çift tırnak reddedilir.
- Tarayıcı modu regresyon testi eklendi.
- Firewall / hosts PowerShell script timeout değeri 60 saniyeye çıkarıldı.
- Timeout davranışı test runner içinde doğrulanır.
- GitHub ikonunun yanına uygulama içi sürüm notları popup butonu eklendi.
- Alt aksiyon çubuğuna Discorder yeniden başlatma butonu eklendi.
- Yeniden başlatma akışı mevcut process kapanana kadar bekleyen helper ile çalışır; single-instance kilidine takılmaz.
- Sağ tanılama alanındaki `Hazır` pasif rozeti `Rapor hazırla` aksiyonuna dönüştürüldü.
- Progress adımları `Hazırlık`, `Kurulum`, `Profil`, `Bağlantı` olarak netleştirildi.
- Hata badge'i `SORUN VAR`, onarım gerektiren durum badge'i `ONARIM GEREKİYOR` olarak güncellendi.
- Tarayıcı modu kilitliyken tooltip artık nedeni açık yazar.

## Kurulum ve Güncelleme

Manuel kurulumda GitHub Releases sayfasından `Discorder-win-x64.zip` indirilebilir. Sürümlü `Discorder-2.1.3-win-x64.zip` asset'i otomatik güncelleme doğrulaması ve release izlenebilirliği için korunur.

Uygulama içi güncellemede Discorder açılışta denetimi arka planda yapar:

1. Yeni sürüm yoksa update düğmesi görünmez.
2. Yeni sürüm varsa `Güncelle` düğmesi görünür.
3. `Güncelle`, denetlenen aynı sürümü indirir, doğrular ve uygular.
4. Hata olursa mevcut sürüm kullanılabilir kalır.

## Güvenlik ve Güven Sınırı

- Discorder otomatik güncelleme paketini GitHub release yolu, GitHub asset digest, SHA-256 dosyası ve update manifesti eşleşmeden uygulamaz.
- Kod imzalama sertifikası yoksa Windows yayıncı kimliği garanti edilemez; bu sürümde güven sınırı GitHub yayın yetkisi ve doğrulama verileridir.
- WireSock fallback MSI varsa hash, Authenticode, yayıncı ve sürüm doğrulaması geçmeden çalıştırılmaz.
- Tarayıcı modu açıkken WireSock kapsamı process seviyesindedir; browser içindeki yalnızca tek Discord sekmesini Windows/WireSock seviyesinde ayırmak garanti edilemez.

## Bilinen Sınırlar

- Public Authenticode sertifikası olmadığı için SmartScreen veya bazı AV ürünleri uyarı gösterebilir.
- Browser cache, DoH, IPv6 veya mevcut açık bağlantılar nedeniyle tarayıcı modu değişikliği yeni oturum gerektirebilir.
- GitHub erişimi olmayan ağlarda update check başarısız olabilir; uygulama mevcut sürümle çalışmaya devam eder.
