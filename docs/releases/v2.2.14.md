# Astral v2.2.14 - Scoped PAC ve hedef testi hotfix

Tarih: 2026-06-18

v2.2.14, v2.2.13 üstüne gelen bağlantı kapsamı hotfix'idir. Amaç, Astral bağlı görünürken seçili web hedeflerinin PAC ayarı eksik kalması veya tarayıcı tarafından okunamaması riskini kapatmaktır.

## Değişenler

- Web hedefleri için scoped PAC ve Windows `AutoConfigURL` durumu `Connected` öncesi canlı doğrulanır.
- Scoped PAC sapması algılanırsa Astral seçili hedef kapsamını bir kez otomatik yeniler; hâlâ doğrulanamazsa bağlı durumuna geçmez.
- PAC ve proxy state dosyaları protected `%ProgramData%` yerine kullanıcı tarafından okunabilir `%LOCALAPPDATA%\Astral\web-proxy` altına taşındı.
- Hızlı hedef testi seçili web hedeflerini Astral.WebProxy üzerinden tek tek ölçer ve başarısız hedefi diğer hedeflerden ayrı gösterir.
- Tanılama paketlerine `webProxyScope.*` alanları eklendi; routing plan ile gerçek Windows PAC durumu ayrı raporlanır.
- Release-notes ikonu sürüm notlarını açacak şekilde düzeltildi; tanılama/onarım/profil temizleme alt bar ikonları aynı görsel sisteme bağlandı.
- Arka plan/video görünürlüğü, panel kontrastı ve hedef kart hizalaması mevcut Astral iki kolon düzeni korunarak iyileştirildi.

## Kullanıcı Etkisi

- Seçili hedefler dışında kalan trafik normal bağlantıda kalmaya devam eder.
- Web hedefleri için genel tarayıcı processleri WireSock `AllowedApps` kapsamına alınmaz; yalnız `Astral.WebProxy.exe` kapsama girer.
- PAC doğrulanamazsa Astral artık yanıltıcı şekilde bağlı görünmez; tanılama paketinde gerçek neden görünür.

## Güvenlik Notu

Web hedefleri HTTPS içeriğini çözmez; CONNECT host allowlist'iyle çalışır. Genel tarayıcı processleri WireSock `AllowedApps` kapsamında değildir. PAC dosyası sadece seçili domainlerde `PROXY`, diğer domainlerde `DIRECT` döndürür.

## Kalan Riskler

- Canlı servis erişimi ISP, servis tarafı değişiklikleri ve yerel güvenlik yazılımlarına göre değişebilir.
- Kurumsal PAC/proxy veya tarayıcı DoH/QUIC ayarları web hedeflerinde ek doğrulama gerektirebilir.
- Uygulama farklı bir Windows hesabının yönetici kimliğiyle çalıştırılırsa HKCU proxy ayarı o hesap için uygulanır; normal kullanımda aynı kullanıcı UAC yükseltmesi beklenir.
