# Astral v2.1.2 - Tek düğmeli güncelleme ve eski web kapsamı stabilizasyonu

Yayın tarihi: 2026-06-15

v2.1.2, v2.1.1 sonrası gelen tanılama kayıtlarından çıkan son kullanıcı karışıklıklarını hedefleyen patch sürümüdür. Ana amaç değişmedi: Astral, Windows üzerinde Discord bağlantısını yönetir; tarayıcı kapsamı yalnızca kullanıcı seçerse dahil edilir.

## Kısa Özet

- Güncelleme akışı tek düğmeye indirildi.
- Yeni sürüm yoksa ana ekranda güncelleme düğmesi gösterilmez.
- Yeni sürüm varsa yalnızca tek `Güncelle` düğmesi görünür ve arka planda bulunan aynı sürüm kurulur.
- Eski web kapsamı kapsamı PATH üzerindeki sahte `chrome.exe` / `msedge.exe` gibi dosyalardan etkilenmeyecek şekilde daraltıldı.
- WireSock istemci dosyası artık imza ve yayıncı kontrolüne ek olarak beklenen sürümle de doğrulanır.
- Updater logları kullanıcıya veya dosyaya yazılmadan önce redaction filtresinden geçirilir.

## Neden Önemli?

Önceki iki aşamalı akışta kullanıcı önce `Güncelle`, sonra `Yükle` görüyordu. Bu teknik olarak güvenliydi ama ürün dili açısından gereksizdi. v2.1.2 ile Astral açılışta güncellemeyi sessizce denetler; gerçek yeni sürüm varsa tek güçlü `Güncelle` düğmesini gösterir.

Eski web kapsamı tarafında WireSock süreç bazlı çalıştığı için kapsamın yanlış executable yollarıyla genişlememesi önemlidir. Bu sürüm, PATH üzerinden gelen sahte tarayıcı adaylarını otomatik kapsama almaz; desteklenen tarayıcı yolları canonical kurulum, App Paths registry ve çalışan güvenilir süreçlerden çıkarılır.

## Değişiklikler

- Ana ekrandaki update alanı tek düğmeye düşürüldü.
- `Yükle` düğmesi ve ikinci kullanıcı adımı kaldırıldı.
- Başlangıçta arka plan update check eklendi.
- Update butonu yalnızca `UpdateAvailable` durumunda görünür.
- Buton tıklaması, daha önce denetlenip kilitlenen update adayını kurar.
- Boşta progress artık sahte `%8` göstermez; gerçek işlem yokken `0% / Hazır` durumundadır.
- Update hazırlık aşamalarının yüzde aralıkları daha tutarlı hale getirildi.
- Browser executable discovery, PATH tabanlı spoof riskini kapsam dışı bıraktı.
- Firewall tarayıcı kapsamı da PATH `Get-Command` adaylarını kullanmaz.
- WireSock CLI sürümü beklenen `1.4.7.1` değeriyle doğrulanır.
- Updater helper logları ve pencere metni redaction filtresiyle yazılır.

## Kurulum ve Güncelleme

Manuel kurulumda GitHub Releases sayfasından `Astral-win-x64.zip` indirilebilir. Sürümlü `Astral-2.1.2-win-x64.zip` asset'i otomatik güncelleme doğrulaması ve release izlenebilirliği için korunur.

Uygulama içi güncellemede Astral açılışta denetimi arka planda yapar:

1. Yeni sürüm yoksa update düğmesi görünmez.
2. Yeni sürüm varsa `Güncelle` düğmesi görünür.
3. `Güncelle`, denetlenen aynı sürümü indirir, doğrular ve uygular.
4. Hata olursa mevcut sürüm kullanılabilir kalır.

## Portable ZIP Notu

Güncelleme, çalışmakta olan `Astral.exe` dosyasının bulunduğu portable klasöre uygulanır. Yeni kurulumlarda klasör adını sabit `Astral` tutmak önerilir. Sürümlü klasörden çalıştırma desteklenir, ancak bu durum tanılama paketinde işaretlenir ve kullanıcı onayı olmadan klasör taşınmaz.

## Güvenlik ve Güven Sınırı

- Astral otomatik güncelleme paketini GitHub release yolu, GitHub asset digest, SHA-256 dosyası ve update manifesti eşleşmeden uygulamaz.
- Kod imzalama sertifikası yoksa Windows yayıncı kimliği garanti edilemez; bu sürümde güven sınırı GitHub yayın yetkisi ve doğrulama verileridir.
- WireSock fallback MSI varsa hash, Authenticode, yayıncı ve sürüm doğrulaması geçmeden çalıştırılmaz.
- Eski web kapsamı açıkken WireSock kapsamı process seviyesindedir; browser içindeki yalnızca tek Discord sekmesini Windows/WireSock seviyesinde ayırmak garanti edilemez.

## Bilinen Sınırlar

- Public Authenticode sertifikası olmadığı için SmartScreen veya bazı AV ürünleri uyarı gösterebilir.
- Browser cache, DoH, IPv6 veya mevcut açık bağlantılar nedeniyle eski web kapsamı kapalıyken web davranışı anlık olarak yanıltıcı görünebilir.
- GitHub erişimi olmayan ağlarda update check başarısız olabilir; uygulama mevcut sürümle çalışmaya devam eder.
