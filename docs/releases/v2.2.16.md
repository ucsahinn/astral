# Astral v2.2.16 - Kanıtlı hedef durumu hotfix

Tarih: 2026-06-18

v2.2.16, v2.2.15 üzerine gelen hedef durumu, güncelleme asset'i ve UI netliği hotfix'idir. Amaç, Astral bağlı durumdayken seçili hedef kartlarının test kanıtı olmadan `Kapsamda` görünmesini engellemek ve tanılama/sürüm notu aksiyonlarını karıştırılmayacak hale getirmektir.

## Değişenler

- Hedef kartları artık global `Connected` durumunu tek başına hedef başarı kanıtı saymaz.
- Bağlantı hazır olduktan sonra seçili hedefler otomatik test kuyruğuna alınır; manuel **Hızlı test** aynı hedef bazlı ölçümü tekrar çalıştırır.
- Her hedef kendi test sonucunu gösterir: test bekliyor, test ediliyor, kapsamda/profil OK veya sorunlu.
- Sağ alt kapsam kartı test sonucu yokken `hedef test bekliyor` yazar; sonuç gelince OK ve sorunlu hedef sayısını ayrı gösterir.
- Seçili hedef kartları cyan/neutral seçim rengine alındı; yeşil başarı vurgusu yalnızca hedef testi veya profil kapsam kanıtı geldikten sonra kullanılır.
- **Rapor Oluştur** tanılama paketini üretir ve klasörü açar; **Sürüm Notları** yalnız uygulama içi release notu penceresini açar.
- Arka plan/video daha görünür kalacak şekilde overlay opaklığı azaltıldı, ana iki kolon layout ve Astral glass/cyan-pink dili korundu.
- Otomatik hedef testi kuyruğu pencere kapanırken veya restart/maintenance akışları sırasında takılmadan güvenli şekilde düşer.
- Release paketinde hem `Astral-win-x64.zip` hem de `Astral-2.2.16-win-x64.zip` yayımlanır; otomatik güncelleme sürümlü ZIP adını bulabilir.

## Kullanıcı Etkisi

- Uygulama artık bir hedefi test etmeden yeşil başarı gibi göstermez.
- Bir hedef sorunluyken diğer hedefler aynı ekranda OK kalabilir; sorun tek kartta görünür.
- Seçili hedefler dışındaki trafik normal bağlantıda kalır. PAC dosyası seçili domainlerde `PROXY`, diğer domainlerde `DIRECT` döndürür.
- Genel tarayıcı processleri WireSock `AllowedApps` kapsamına girmez; web hedefleri yalnız `Astral.WebProxy.exe` üzerinden kapsama alınır.

- `powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify.ps1`

## Güvenlik Notu

Web hedefleri HTTPS içeriğini çözmez; CONNECT host allowlist'iyle çalışır. Tarayıcı processleri WireSock profiline eklenmez. Hedef testi, web hedefleri için yerel scoped proxy üzerinden `CONNECT host:443` ölçer ve içerik okumaz.

## Kalan Riskler

- Canlı hedef erişimi ISP, servis tarafı değişiklikleri ve yerel güvenlik yazılımlarına göre değişebilir.
- Uygulama hedeflerinde `Profil OK`, uygulamanın WireSock kapsamında olduğunu gösterir; üçüncü taraf uygulamanın kendi login/servis durumunu garanti etmez.
- Tarayıcı DoH/QUIC, kurumsal PAC/proxy ve güvenlik yazılımları web hedeflerinde ek yerel doğrulama gerektirebilir.
