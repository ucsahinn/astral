# Astral v2.2.5 - Bağlantı koruması ve hedef kartları hotfix

Tarih: 2026-06-17

v2.2.5, v2.2.4 tanılama paketlerinden görülen bağlantı koruması temizleme hatasını ve ana ekrandaki hedef kartı okunurluğunu hedefleyen hotfix sürümüdür.

## Değişenler

- Bağlantı başlarken eski/stale Astral veya legacy firewall kuralı temizliği daha toleranslı hale getirildi.
- `Disable` access-lock hatalarında artık yalnız `PowerShell exit code 1` değil, script fazı ve stdout/stderr kuyruğu da tanılamaya girer.
- Bağlantı kapalıyken etkinleşen hedef koruması artık sadece sabit Discord listesini değil, seçili hedef planındaki exact domainleri kullanır.
- Hosts dosyası Astral marker bloğu yazılıp kaldırılırken read-only attribute gibi basit Windows dosya durumları toparlanır.
- Eski marka klasöründen çalışan Astral, tanılama paketinde artık bunu “sabit Astral klasörü” olarak yanlış raporlamaz.
- Uzun bağlantı hataları ana ekranda kırpılmasın diye status başlığı ve detay metni ayrıldı.
- Hedef kartları daha okunur hale getirildi; ikon rozeti, scope etiketi ve kilitli kart görünümü güçlendirildi.

## Güvenlik ve Kapsam

- Genel tarayıcı süreçleri WireSock `AllowedApps` kapsamına alınmaz.
- Web hedefleri için seçili domainler PAC/WebProxy kapsamından geçer; TLS içeriği çözülmez.
- Loglarda paket içeriği, cookie, token, WireGuard private key veya credential yazılmaz.
- Üçüncü taraf resmi marka dosyaları bu sürüme gömülmedi; kartlar lisans riski oluşturmayan brand-color fallback marklarla kalır.

## Güncelleme Notu

Manuel kurulumda GitHub Releases sayfasından `Astral-win-x64.zip` indirilebilir. Otomatik güncelleme için sürümlü `Astral-2.2.5-win-x64.zip` korunur.

Kod imzalama sertifikası bu build ortamında yapılandırılmadıysa paket imzasızdır. Güncelleme güven sınırı GitHub release yolu, asset digest, SHA-256 dosyası ve manifest eşleşmesidir.
