# Astral v2.1.1 - Bağlantı hazırlığı ve eski web kapsamı hotfix

Tarih: 2026-06-14

v2.1.1, v2.1.0 sonrası gelen tanılama paketlerinden çıkan üç gerçek kullanıcı akışını düzeltir: eski ayarlarda açık kalmış Eski web kapsamı, WireSock kurucusuna erişilemeyen ilk kurulumlar ve Discord'un ilk denemede bulunamayıp ikinci denemede açılması.

## Kısa Özet

- Eski ayarlarda `BrowserAccessEnabled=true` kalmışsa Eski web kapsamı güvenli varsayılana, yani kapalı duruma taşınır.
- Kullanıcı Eski web kapsamınu daha sonra bilinçli açarsa tercih korunur.
- WireSock hazırlığı artık doğrulanmış yerel MSI paketi bulursa resmi indirme yapmadan kullanabilir.
- Release build, yerel `vendor\wiresock` altında doğru SHA-256 değerine sahip resmi WireSock MSI bulunursa pakete `installers` klasörü altında ekleyebilir.
- Discord otomatik açma akışı, klasik Squirrel kurulumlarda güvenli `Update.exe --processStart Discord.exe` fallback planını kullanabilir.
- Tanılama loglarında görülen "ilk deneme bulamadı, ikinci deneme açtı" davranışı azaltıldı.

## Ne Değişti?

- `AppSettingsStore`, Eski web kapsamı tercih versiyonunu saklar ve eski implicit açık değeri bir kere kapalıya migrate eder.
- Uygulama başlangıcı migration olursa `settings.browserAccess.migrated` olayı yazar.
- `WireSockBootstrapper`, indirmeden önce app klasörü, `installers`, `WireSock` ve Astral installer data klasörlerinde doğrulanmış MSI arar.
- Yerel WireSock MSI yalnızca beklenen SHA-256 ve installer doğrulaması geçerse kullanılır.
- `scripts\build-release.ps1`, `vendor\wiresock\wiresock-vpn-client-x64-1.4.7.1.msi` dosyası varsa hash doğrulayıp publish çıktısına kopyalar. Dosya yoksa mevcut build davranışı değişmez.
- `WindowsDiscordProcessInspector`, Discord app klasörü yakalanamazsa güvenli updater launch planı oluşturabilir.

## Neden Önemli?

Eski web kapsamı açıkken WireSock process seviyesinde çalışır. Bu yüzden Brave, Chrome veya Edge profile girerse tarayıcıdaki sadece Discord sekmesini değil, aynı tarayıcı prosesinin başka trafiğini de etkileyebilir. v2.1.1 eski makinelerde açık kalmış bu davranışı kapalıya taşıyarak "bağlıyken internet yavaşlıyor" riskini azaltır.

WireSock yüklü olmayan PC'lerde ilk bağlantı resmi kurucu indirmesine bağlıydı. Ağ zayıfsa veya `wiresock.net` bağlantısı zaman aşımına düşerse bağlantı başlamadan kalıyordu. v2.1.1 doğrulanmış yerel kurucu fallback'i ekler; release paketine resmi MSI dahil edilirse ilk kurulum daha dayanıklı olur.

Discord tarafında bazı makinelerde ilk denemede kurulum bulunamayabiliyor, sonra kullanıcı tekrar deneyince Discord açılıyordu. Updater fallback planı bu ilk deneme kırılganlığını azaltır.

## Kurulum ve Güncelleme Notları

1. Uygulama içinden `Güncelle` butonuna basın.
2. Yeni sürüm bulunursa `Yükle` ile v2.1.1 paketini kurun.
3. Güncelleme tamamlanınca Astral yeniden açılır.

Manuel kurulumda GitHub Releases sayfasından `Astral-win-x64.zip` indirilebilir. Sürümlü `Astral-2.1.1-win-x64.zip` asset'i otomatik güncelleme doğrulaması ve release izlenebilirliği için korunur.

## Portable ZIP Davranışı

- Yeni kurulumlarda ZIP'i `Astral` gibi sabit adlı bir klasöre çıkarmak önerilir.
- Sürümlü klasörden çalıştırma desteklenir, ancak tanılamada görünür.
- Güncelleme mevcut çalışan klasörü hedefler ve doğrulanmış paket dışında dosya uygulamaz.

## Güvenlik ve Gizlilik

- WireSock MSI fallback'i hash ve installer doğrulaması geçmeden kullanılmaz.
- Discord updater fallback'i yalnızca trusted Discord/Update.exe yolu üzerinden oluşturulur.
- Debug tanılama secrets-safe kalır; token, cookie, private key, komut satırı veya paket yakalama içermez.
- Bu build Authenticode imzalı değildir; SmartScreen veya bazı AV uyarıları görülebilir.

## Bilinen Sınırlar

- Release paketinde WireSock MSI yoksa ilk kurulum hâlâ resmi indirme bağlantısına ihtiyaç duyar.
- Eski web kapsamı kullanıcı tarafından açılırsa browser süreçleri profile dahil olabilir; bu Windows/WireSock process kapsamıdır.
- Sertifika olmadığı için AV/SmartScreen itibarı garanti edilemez.
