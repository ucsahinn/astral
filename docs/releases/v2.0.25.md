# Discorder v2.0.25 - Tanılama metrikleri hotfix

Tarih: 2026-06-13

v2.0.25, v2.0.24 sonrasında gerçek kullanıcı tanılama paketinde görülen negatif `managedMemoryBytes` ve `managedMemoryMb` değerlerini düzeltir. Bağlantı ve update akışı aynı kalır; bu sürüm diagnostic çıktısının güvenilirliğini artırır.

## Kısa Özet

- Runtime metrikleri artık sıfır altına düşecek şekilde raporlanmaz.
- `health.json` ve `runtime.json` içindeki bellek sayaçları güvenli şekilde normalize edilir.
- Tanılama özeti negatif managed memory göstermeyecek hale getirildi.
- Testler `health.json` ve paket içindeki `runtime.json` metriklerinin negatif olmadığını doğrular.

## Ne Değişti?

- `DiscorderDiagnostics`, runtime sayaçlarını rapora yazmadan önce negatif değerlere karşı korur.
- Working set, private memory, managed memory, GC heap ve fragmented heap sayaçları `0` altına inemez.
- Diagnostic bundle testleri runtime metriklerini JSON seviyesinde doğrular.

## Neden Önemli?

Tanılama paketi destek ve hata çözümü için kaynak kabul ediliyor. Negatif bellek değeri gerçek bir uygulama durumu değil; kullanıcıyı ve debug sürecini yanıltıyordu. Bu hotfix tanılama çıktısını sakin, okunabilir ve doğru hale getirir.

## Kurulum ve Güncelleme Notları

1. GitHub Releases sayfasından `Discorder-2.0.25-win-x64.zip` arşivini indirin.
2. Yanındaki `Discorder-2.0.25-win-x64.sha256.txt` dosyasıyla arşivi kontrol edin.
3. ZIP'i sabit bir klasöre çıkarın.
4. `Discorder.exe` dosyasını çalıştırın.

Uygulama içi güncellemede `Güncelle` önce yeni sürümü denetler. `Yükle` göründüğünde indirilen paket GitHub release asset bilgisi, SHA-256 dosyası ve update manifestiyle eşleşmeden uygulanmaz.

## Portable ZIP Davranışı

- Güncelleme mevcut çıkarılmış Discorder klasörünü hedefler.
- Paket önce korumalı staging alanına indirilir.
- ZIP özeti, `.sha256.txt` dosyası ve update manifest doğrulanmadan dosyalar uygulanmaz.
- Update helper doğru `Discorder.exe` dosyasını yeniden başlatır.
- Hata olursa mevcut çalışan klasör bozulmadan kalır.

## Güvenlik ve Güven Sınırları

- WireSock kurucusu Authenticode, yayıncı, sürüm ve SHA-256 kontrollerinden geçer.
- `wgcf` aracı SHA-256 ile doğrulanır ve boyut sınırıyla indirilir.
- Uygulama güncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden geçer.
- Diagnostic bundle log kopyalama sırasında token, private key, cookie ve authorization benzeri değerler redakte edilir.
- Bu build Authenticode imzalı değildir; Windows SmartScreen uyarısı görülebilir.

## Bilinen Sınırlar

- Bu sürüm bağlantı motorunu değiştirmez; v2.0.24'teki Discord yenileme ve WireSock `AllowedApps` davranışı korunur.
- GitHub erişimi olmayan ağlarda güncelleme denetimi başarısız olabilir; mevcut sürüm çalışmaya devam eder.
