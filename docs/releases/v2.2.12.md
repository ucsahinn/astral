# Astral v2.2.12 - Live smoke ve transparent readiness hotfix'i

Tarih: 2026-06-18

v2.2.12, v2.2.11 üstüne gelen küçük ama önemli doğrulama hotfix'idir. Uygulama davranışı aynı hedef odaklı modelde kalır; WireSock transparent mode readiness düzeltmesi korunur ve canlı smoke testi dış ağ/ISS durumunu yanlış release bloklayıcısı saymayacak şekilde güncellenir.

## Değişenler

- WireSock varsayılan transparent mode'da çalışırken `wiresock-adapter-inactive` tek başına hata sayılmaz.
- WireSock süreci birkaç readiness kontrolü boyunca sağlıklı kalırsa tanılama `tunnelReadiness=transparent-process-running` ve `wireSockMode=transparent` bilgisini yazar.
- WireSock süreci erken kapanırsa bağlantı hata sayılır; çıkış durumu `wireSockProcessExited` ve `wireSockProcessExitCode` alanlarıyla görünür kalır.
- Profil üretimi WireSock uyumlu `#@ws:AllowedApps` direktifini kullanır ve eski/geniş `AllowedApps` satırlarını temizler.
- Web hedeflerinde genel tarayıcı süreçleri profile eklenmez; seçili domainler yalnız `Astral.WebProxy.exe` üzerinden kapsamlanır.
- WebProxy başlatılmadan önce aynı executable path'e ait eski orphan `Astral.WebProxy` süreçleri temizlenir.
- Live smoke scriptinde `discord.com:443` doğrudan ağ probu bilgi amaçlı bırakıldı; ISS/servis durumuna bağlı bu prob release başarısını tek başına düşürmez.

## Kullanıcı Etkisi

- Astral, WireSock transparent mode'da sanal adaptör görünmediği için bağlantıyı gereksiz yere başarısız saymaz.
- Seçili hedefler hâlâ dar kapsamlı kalır; tüm PC, tüm tarayıcı veya sistem geneli VPN davranışı oluşmaz.
- Canlı smoke sonucu artık hedef kapsamı, profil, WebProxy, WireSock lifecycle ve cleanup kanıtını esas alır.

## Güvenlik Notu

Web hedefleri HTTPS içeriğini çözmez; CONNECT host allowlist'iyle çalışır. Genel tarayıcı processleri WireSock `#@ws:AllowedApps` kapsamında değildir. Paket imzası yoksa güncelleme güven sınırı GitHub release yolu, asset digest, SHA-256 dosyası ve manifest eşleşmesidir.

## Kalan Riskler

- Canlı servis erişimi ISS, servis tarafı değişiklikleri ve yerel güvenlik yazılımlarına göre değişebilir.
- Kurumsal PAC/proxy veya tarayıcı DoH/QUIC ayarları web hedeflerinde ek doğrulama gerektirebilir.
- Kod imzalama sertifikası yapılandırılmadıysa Windows SmartScreen uyarısı görülebilir.
