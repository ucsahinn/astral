# Astral v2.2.6 - WebProxy tünel logu ve logo paneli hotfix

Tarih: 2026-06-17

v2.2.6, v2.2.5 sonrası tanılama paketlerinde görülen WebProxy başlatma hatasını ve hedef paneli görsel sıkışmasını hedefleyen hotfix sürümüdür.

## Değişenler

- WireSock ve Astral.WebProxy aynı `tunnel.log` dosyasını eş zamanlı yazabilir hale getirildi.
- `tunnel.log` satırları artık süreç adını taşır; WireSock ve WebProxy çıktıları tanılamada daha net ayrılır.
- Web hedefleri seçiliyken `Astral.WebProxy` başlatma aşamasında görülen `The process cannot access the file ... tunnel.log` hatası giderildi.
- Tünel kapsamı alanındaki görünür açıklama metinleri kaldırıldı; alan desteklenen ürünlerin logo odaklı seçim yüzeyi olarak sadeleştirildi.
- Hedef tile padding ve seçili tik boyutu iyileştirildi; ürün markları küçük alanda daha temiz render edilir.
- GitHub footer ikonu kenara dayanmayan güvenli viewport ile çizilir.

## Güvenlik ve Kapsam

- Genel tarayıcı süreçleri WireSock `AllowedApps` kapsamına alınmaz.
- Web hedefleri için yalnızca seçili domainler PAC/WebProxy kapsamından geçer; TLS içeriği çözülmez.
- Loglarda cookie, token, WireGuard private key veya credential yazılmaz.
- Üçüncü taraf resmi marka dosyaları bu sürüme gömülmedi; hedef kartları lisans riski oluşturmayan marka renkli fallback marklarla kalır.

## Güncelleme Notu

Manuel kurulumda GitHub Releases sayfasından `Astral-win-x64.zip` indirilebilir. Otomatik güncelleme için sürümlü `Astral-2.2.6-win-x64.zip` korunur.

Kod imzalama sertifikası bu build ortamında yapılandırılmadıysa paket imzasızdır. Güncelleme güven sınırı GitHub release yolu, asset digest, SHA-256 dosyası ve manifest eşleşmesidir.
