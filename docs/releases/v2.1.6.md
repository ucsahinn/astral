# Discorder v2.1.6 - Discord arka plan readiness hotfix'i

Yayın tarihi: 2026-06-17

v2.1.6, v2.1.5 sonrası gerçek tanılama bundle'ında görülen kalan Discord readiness false-negative hatasını kapatan patch sürümdür. Discorder artık güvenilir Discord süreçleri arka planda çalışıyorsa, Windows görünür ana pencere handle'ı vermese bile bağlantıyı gereksiz yere `DiscordRestartRequired` durumuna düşürmez.

## Kısa Özet

- v2.1.5 ile gelen Discord updater recovery ve güvenli tray çıkış davranışı korunur.
- Güvenilir `Discord.exe` süreçleri ayakta kaldığında pencere görünmese bile bağlantı hazır kabul edilir.
- Sadece tek updater penceresi görülen durum hâlâ hazır sayılmaz; manuel aksiyon mesajı korunur.
- Yeni regresyon testi, updater penceresi + çoklu güvenilir arka plan Discord süreci varyantını kapsar.

## Neden Önemli?

Bazı makinelerde Discord, updater toparlandıktan sonra ana pencere handle'ını Windows'a hemen vermeden çoklu Electron süreçleriyle arka planda çalışmaya devam edebiliyor. v2.1.5 bu durumda tüneli ve WireSock sürecini başlatmış, Discord süreçlerini de doğrulamış olmasına rağmen görünür pencere kanıtı gelmediği için bağlantıyı manuel aksiyona düşürebiliyordu.

Yeni davranış daha doğru ayrım yapar:

- Görünür ana Discord penceresi varsa bağlantı hazırdır.
- Görünür pencere yok ama güvenilir Discord executable'ına ait süreçler çalışıyorsa bağlantı arka planda hazırdır.
- Sadece updater penceresi görülen tek süreç durumu hâlâ tamamlanmış Discord oturumu sayılmaz.

## Değişiklikler

- Windows Discord readiness sınıflandırmasına `TrustedBackgroundProcess` sonucu eklendi.
- `VerifyReadyAsync`, güvenilir arka plan Discord süreci görüldüğünde başarılı döner.
- `LaunchDiscordAsync`, Discord arka planda süreçlerle hazır olduğunda launch sonucunu başarılı kabul eder.
- Updater-only hata dalı korunur; güvenilir süreç kanıtı olmadan bağlantı hazır sayılmaz.
- Core regression testlerine arka plan süreç readiness sınıflandırması eklendi.

## Kurulum ve Güncelleme

Manuel kurulumda GitHub Releases sayfasından `Discorder-win-x64.zip` indirilebilir. Sürümlü `Discorder-2.1.6-win-x64.zip` asset'i otomatik güncelleme doğrulaması ve release izlenebilirliği için korunur.

Uygulama içi güncellemede Discorder açılışta denetimi arka planda yapar:

1. Yeni sürüm yoksa update düğmesi görünmez.
2. Yeni sürüm varsa `Güncelle` düğmesi görünür.
3. `Güncelle`, denetlenen aynı sürümü indirir, doğrular ve uygular.
4. Hata olursa mevcut sürüm kullanılabilir kalır.

## Güvenlik ve Güven Sınırı

- Discorder otomatik güncelleme paketini GitHub release yolu, GitHub asset digest, SHA-256 dosyası ve update manifesti eşleşmeden uygulamaz.
- Kod imzalama sertifikası yoksa Windows yayıncı kimliği garanti edilemez; bu sürümde güven sınırı GitHub yayın yetkisi ve doğrulama verileridir.
- WireSock fallback MSI varsa hash, Authenticode, yayıncı ve sürüm doğrulaması geçmeden çalıştırılmaz.
- Tarayıcı modu açıkken WireSock kapsamı process seviyesindedir; browser içindeki yalnızca tek Discord sekmesini Windows/WireSock seviyesinde ayırmak garanti edilemez.

## Bilinen Sınırlar

- Public Authenticode sertifikası olmadığı için SmartScreen veya bazı AV ürünleri uyarı gösterebilir.
- Discord updater pencere sıralaması işletim sistemi ve Discord sürümüne bağlıdır; updater-only durumunda manuel aksiyon mesajı korunur.
- Browser cache, DoH, IPv6 veya mevcut açık bağlantılar nedeniyle tarayıcı modu değişikliği yeni oturum gerektirebilir.
- GitHub erişimi olmayan ağlarda update check başarısız olabilir; uygulama mevcut sürümle çalışmaya devam eder.
