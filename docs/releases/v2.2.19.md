# Astral v2.2.19 - Probe DNS ve WebProxy dayanıklılık adayı

Tarih: 2026-06-18

> Not: v2.2.19 yayınlandıktan sonra bazı makinelerde WireSock'un yeni `astral-scoped.conf` runtime profil adıyla adaptörü ayağa kaldıramadığı görüldü. Bu bağlantı regresyonu v2.2.20 hotfix ile giderildi; güncel kullanıcılar v2.2.23 veya üstünü kullanmalıdır.

v2.2.19, v2.2.18 tanı paketlerinde görülen bağlantı kapsamı ve yanıltıcı rota testi sonuçlarını düzeltmek için hazırlanmış aday sürümdür. Loglar WireSock ve Astral.WebProxy lifecycle'ının ayağa kalktığını, kalan kullanıcı etkisinin eski profil adı, web hedeflerinde seçili-domain kilidi, API probe timeout'ları ve restart cleanup davranışı tarafında olduğunu gösterdi.

## Değişenler

- Rota kapsamı ile canlı probe hostları ayrıldı. Route-only domainler proxy allowlist içinde kalır, fakat otomatik rota testi DNS'te çözülemeyen veya liveness için güvenilir olmayan hostlara takılmaz.
- Discord, Azar, LiVU ve Tango için probe listeleri stabil temsilci hostlara taşındı.
- Astral.WebProxy upstream DNS çözümlemesi hosts lock veya sistem DNS/PAC etkisiyle başarısız olduğunda önce proxy kullanmadan Cloudflare DoH JSON fallback dener; DoH/TLS katmanı kullanılamazsa `1.1.1.1` ve `8.8.8.8` üzerinden ham UDP DNS fallback uygular.
- Astral.WebProxy upstream kuramazsa tarayıcı tarafını sessiz bekletmek yerine `HTTP/1.1 502 Bad Gateway` döndürür ve yalnız sanitize edilmiş host adını loglar.
- DoH, UDP DNS veya upstream timeout/hata durumları WebProxy görevini düşürmeden tanılanabilir proxy cevabına çevrilir.
- Sağ hedef panelindeki manuel `Hızlı Test` aksiyonu kaldırıldı. Kullanıcıya hedef login garantisi yerine yalnız seçili kapsamın bağlantıya dahil edildiği gösterilir.
- Hedef kartları API probe timeout'larını kırmızı ürün hatasına veya `Onar` sebebine çevirmeden `Kapsam hazır` / `Kapsam aktif` durumunu gösterir.
- Uygulama açılışında güncelleme denetimi bağlantı kilidi hazırlığından bağımsız başlatılır; yeni sürüm varsa güncelleme butonu görünür.
- Ana pencere 1280x720 sabit viewport'a alındı; çözünürlük/pencere yeniden boyutlandırma kaynaklı layout kayması engellendi ve 1366x768 sınıfı ekranlarda taşma riski düşürüldü.
- Yeniden başlatma akışı kapanış temizliği zaman aşımına uğrasa bile takılı kalmaz; tanılamaya uyarı yazıp restart helper'ı best-effort başlatır.
- Alt bardaki tanılama aksiyonu `Rapor Oluştur` olarak ayrıştırıldı; `Sürüm Notları` yalnız release notu penceresini açar.
- `Onar` aksiyonu artık sorun algılanmadığında pasif kalır ve disabled tooltip ile nedenini açıklar; API probe timeout'u onarım sebebi yapılmaz, gerçek bağlantı hata state'i onarımı etkinleştirir.
- Tarihsel not: v2.2.19 WireSock profilini `astral-scoped.conf` adıyla üretmeyi denedi; bu davranış bazı makinelerde adaptörün ayağa kalkmamasına yol açtığı için v2.2.20 ile runtime profil adı yeniden `discord.conf` uyumluluk yoluna döndürüldü.
- Yeni firewall/hosts kilidi `Astral.BlockTargetDomains` ve `Astral hedef kilidi` adlarını kullanır; eski `Astral.BlockDiscordDomains` ve eski marka marker'ları kaldırma/disable sırasında temizlenir.
- Tünel açıkken tarayıcı düz bağlantı koruması artık yalnız Discord varsayılanlarına düşmez; Wattpad, Blogspot ve diğer seçili web hedeflerinin domainleriyle browser direct kaçışı kilitler.
- Video arka plan korunarak overlay ve panel opaklıkları hafifletildi; hedef paneli ve alt aksiyonlar aynı paletle daha okunur hale getirildi.

## Kullanıcı Etkisi

- Bağlantı açıkken seçili hedefler artık API probe timeout'u yüzünden yanlış `Sorunlu` görünmez.
- Sağ hedef paneli kullanıcıya yalnız kapsamın hazır/aktif olduğunu gösterir; gerçek hedef uygulama veya web sitesini kullanıcı kendisi açar.
- Web hedefleri yine yalnız seçili domainlerde proxy kullanır; diğer trafik DIRECT kalır.
- Genel tarayıcı süreçleri WireSock `AllowedApps` içine eklenmez.
- Upstream/DNS problemi olduğunda tanı paketlerinde daha açık WebProxy kanıtı oluşur.
- Güncelleme varsa açılıştan sonra görünür buton tekrar gelir; güncel sürümde buton gizli kalır.

## Güvenlik Notu

Web hedefleri HTTPS içeriğini çözmez; CONNECT host allowlist'iyle çalışır. DoH/UDP DNS fallback yalnız hostu IP'ye çözmek için kullanılır, TLS içeriğini açmaz ve sertifika kurmaz. WebProxy logları path, query, header, cookie, token, body veya kullanıcı içeriği yazmaz.

Bu build ortamında kod imzalama sertifikası yapılandırılmadığı için ZIP içindeki `Astral.exe`, `Astral.Updater.exe` ve `Astral.WebProxy.exe` imzasızdır. SHA-256 dosyaları release asset doğrulaması için üretilmiştir.

## Kalan Riskler

- Gerçek servis login akışı servis tarafı değişiklikleri, captcha, hesap güvenlik kontrolleri ve üçüncü taraf auth sağlayıcılarına bağlı olabilir.
- Bu sürüm rota ve proxy taşıma kanıtını güçlendirir; kullanıcı hesabıyla gerçek giriş yapmayı otomatik olarak test etmez.
- DoH ve UDP DNS fallback yolları aynı anda erişilemezse WebProxy bunu 502 olarak bildirir; seçili hedefin çalışması yine kullanılabilir upstream DNS ve hedef servis erişimine bağlıdır.
