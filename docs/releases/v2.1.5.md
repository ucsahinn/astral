# Discorder v2.1.5 - Discord updater ve güvenli çıkış hotfix'i

Yayın tarihi: 2026-06-17

v2.1.5, v2.1.4 sonrası gerçek kullanımda görülen iki kritik akışı kapatan patch sürümdür. Discord updater penceresinde takılma durumunda bağlantı daha kontrollü doğrulanır; Discorder bildirim alanındaki `Çıkış` komutu ise arka plan ayarı açık olsa bile aktif tüneli güvenli kapatmadan süreç bırakmaz.

## Kısa Özet

- Discord updater veya geciken ana pencere durumunda bağlantı hemen manuel aksiyona düşmeden sınırlı tekrar doğrulanır.
- Tünel yenilemesi sonrasında updater hâlâ görünürse Discord hazır olma durumu tünel üstünden tekrar kontrol edilir.
- Bildirim alanındaki `Çıkış`, arka planda çalışma ayarı açık olsa bile güvenli shutdown ve controller cleanup yolundan geçer.
- `windir` veya `SystemRoot` eksik başlatma ortamlarında WPF açılışı sertleştirildi.
- v2.1.4 tooltip ve tanılama paneli düzeni korunur.

## Neden Önemli?

Discord güncelleme penceresi bazı makinelerde ana Discord penceresinden önce görünebiliyor. Önceki sürüm bu durumda tünel geri geldikten sonra yeterince tekrar doğrulama yapmadan kullanıcıya manuel kapatma mesajı verebiliyordu. v2.1.5, updater ve görünmeyen pencere durumlarını ayrı sınıflandırır ve sadece sınırlı, ölçülebilir hazır olma kontrolleri yapar.

Arka planda çalışma ayarı açıkken pencereyi kapatmak hâlâ bilinçli olarak bildirim alanına saklar. Fakat bildirim alanındaki `Çıkış` artık farklıdır: uygulama gerçekten kapanmak istendiğini anlar, aktif WireSock bağlantısını ve controller kaynaklarını temizler, sonra pencereyi kapatır. Böylece tek-instance kilidi veya arka planda kalan süreç yüzünden yeni sürümün açılamaması engellenir.

## Değişiklikler

- `WindowNotVisible` Discord restart failure türü eklendi.
- Discord readiness kontrolü updater ve görünmeyen pencere durumları için en fazla iki denemeyle sınırlandı.
- Discord updater recovery sonrası tünel yeniden başlatıldığında hazır olma kontrolü tekrar çalıştırılır.
- Tray `Çıkış` yolu `_allowClose` ile cleanup atlamaz; güvenli kapanış dalını kullanır.
- Windows presentation ortamı, eksik `windir` / `SystemRoot` process değişkenlerini Windows klasöründen tamamlar.
- Core ve Windows smoke testlerine updater gecikmesi, görünmeyen pencere ve tray çıkış cleanup regresyonları eklendi.

## Kurulum ve Güncelleme

Manuel kurulumda GitHub Releases sayfasından `Discorder-win-x64.zip` indirilebilir. Sürümlü `Discorder-2.1.5-win-x64.zip` asset'i otomatik güncelleme doğrulaması ve release izlenebilirliği için korunur.

Uygulama içi güncellemede Discorder açılışta denetimi arka planda yapar:

1. Yeni sürüm yoksa update düğmesi görünmez.
2. Yeni sürüm varsa `Güncelle` düğmesi görünür.
3. `Güncelle`, denetlenen aynı sürümü indirir, doğrular ve uygular.
4. Hata olursa mevcut sürüm kullanılabilir kalır.

## Güvenlik ve Güven Sınırı

- Discorder otomatik güncelleme paketini GitHub release yolu, GitHub asset digest, SHA-256 dosyası ve update manifesti eşleşmeden uygulamaz.
- Kod imzalama sertifikası yoksa Windows yayıncı kimliği garanti edilemez; bu sürümde güven sınırı GitHub yayın yetkisi ve doğrulama verileridir.
- WireSock fallback MSI varsa hash, Authenticode, yayıncı ve sürüm doğrulaması geçmeden çalıştırılmaz.
- Tarayıcı modu açıkken WireSock kapsamı process seviyesindedir; browser içindeki yalnızca tek Discord sekmesini Windows/WireSock seviyesinde ayırmak garanti edilemez.

## Bilinen Sınırlar

- Public Authenticode sertifikası olmadığı için SmartScreen veya bazı AV ürünleri uyarı gösterebilir.
- Discord updater pencere sıralaması işletim sistemi ve Discord sürümüne bağlıdır; v2.1.5 bu yarışı sınırlı tekrar doğrulamayla toparlar, sınırsız bekleme yapmaz.
- Browser cache, DoH, IPv6 veya mevcut açık bağlantılar nedeniyle tarayıcı modu değişikliği yeni oturum gerektirebilir.
- GitHub erişimi olmayan ağlarda update check başarısız olabilir; uygulama mevcut sürümle çalışmaya devam eder.
