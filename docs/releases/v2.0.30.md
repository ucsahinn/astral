# Astral v2.0.30 - seçili hedef baglanti stabilizasyonu

Tarih: 2026-06-14

v2.0.30, Astral'in ana urun vaadini daha net ve olculebilir hale getirir: `BAGLAN` denildiginde varsayilan kapsam yalnizca Discord masaustu uygulamasidir. Tarayici Discord trafigi artik ilk kurulumda otomatik dahil edilmez; kullanici isterse Eski web kapsamı ile ayrica acar.

## Kisa Ozet

- Eski web kapsamı yeni kurulumlarda varsayilan olarak kapali gelir.
- Eski web kapsamı aciksa desteklenen tarayicilar yalnizca bulunan executable yollariyla kapsama eklenir.
- Bare `chrome.exe`, `msedge.exe` gibi genis tarayici process adlari WireSock `AllowedApps` listesine eklenmez.
- WireSock config ozeti, config SHA-256 degeri ve etkili routing kapsami tanilamaya eklenir.
- Astral'in baslattigi WireSock sureci ownership marker ile takip edilir; crash/restart sonrasi yetim surec daha guvenli temizlenir.
- `errors.log`, `tunnel.log` ve unhandled exception loglari yazim aninda redakte edilir.
- WireSock kapatma akisi bounded wait kullanir; kapanis uzun sure pipe/log beklemesine takilmaz.
- UI canli kart yenilemesindeki gereksiz cift refresh kaldirildi.

## Ne Degisti?

- `AppSettingsStore`, yeni kurulumlarda tarayici erisimini `false` kabul eder.
- `DiscordAppScope`, eski web kapsamı acikken sadece gercek bulunan tarayici executable path'lerini profile ekler.
- `DiscordTunnelController`, uretilen profil hash'ini ve routing ozetini status, health ve event loglarina yazar.
- `DiscordTunnelController`, Astral'in sahip oldugu WireSock PID/start-time marker'ini kaydeder ve yeniden acilista bu owned orphan sureci temizlemeye calisir.
- `ProcessLauncher`, kill sonrasi WireSock surecinin cikmasini sinirli sure bekler; log pump kapanisi timeout olursa uygulama kapanisini gereksiz uzatmaz.
- Kalici hata/tunel loglari bundle asamasini beklemeden redakte edilir.
- Testler eski web kapsamınun kapali varsayilmasini, tarayici path kapsamlarini ve redakte edilmis hata logunu kilitler.

## Neden Onemli?

Kullanici icin asil beklenti basittir: Astral acildiginda genel bilgisayar interneti yavaslamamali, `BAGLAN` sadece Discord baglantisini yonetmeli, tarayici dahil edilecekse bu kullanicinin net tercihi olmali.

v2.0.30 bu davranisi varsayilan seviyede daha dar hale getirir. Tanilama dosyalari da artik "hangi mod acikti, profile hangi kapsam girdi, WireSock hangi config ile calisti" sorularina daha net cevap verir.

- Paket once korumali staging alanina indirilir.
- ZIP ozeti, `.sha256.txt` dosyasi ve update manifest dogrulanmadan dosyalar uygulanmaz.
- Update helper dogru `Astral.exe` dosyasini yeniden baslatir.
- Eski surumlu klasor adlari tanilamada raporlanir; kullanici onayi olmadan klasor tasinmaz veya silinmez.

## Guvenlik ve Guven Sinirlari

- Eski web kapsamı kapaliyken tarayici process adlari WireSock kapsamindan cikarilir.
- Eski web kapsamı acikken kapsam process seviyesindedir; browser icinde yalnizca Discord tab'ini ayirmak Windows/WireSock seviyesinde garanti edilemez.
- Loglar kalici yazimda redakte edilir; token, private key ve kullanici path'i gibi hassas veriler destek paketinde de maskelenir.
- App-created WireSock sureci marker ile takip edilir; rastgele WireSock surecleri genis sekilde oldurulmez.
- Bu build Authenticode imzali degildir; Windows SmartScreen veya bazi AV uyarilari gorulebilir.

## Bilinen Sinirlar

- Sertifika olmadigi icin AV/SmartScreen itibari tum vendorlarda garanti edilemez.
- WireSock `AllowedApps` semantigi process/app seviyesindedir; eski web kapsamı acildiginda browser process'inin Discord disi sekmeleri de ayni process icinde olabilir.
- Kullanici manuel olarak eski surumlu klasorden calistiriyorsa tanilama bunu raporlar; klasor tasima/silme destructive sayildigi icin otomatik yapilmaz.
