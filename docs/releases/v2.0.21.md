# Discorder v2.0.21 - Bellek, tanilama ve premium UI hotfix

Tarih: 2026-06-13

v2.0.21, v2.0.20 uzerine gelen performans ve gorsel polish patch surumudur. Amac, Discorder bagli degilken gereksiz medya maliyetini azaltmak, tanilama paketlerinde RAM/GC bilgisini net gostermek ve ana ekranin premium hissini fonksiyon kaybetmeden guclendirmektir.

## Kisa Ozet

- Baglanti yokken arka plan videosu otomatik durdurulur ve kaynak serbest birakilir.
- Minimize, tray ve kapatma yollarinda medya maliyeti daha agresif sekilde kesilir.
- Arka plan videosu public build'de runtime CDN fallback yapmaz; yerel asset yoksa statik premium yuzey kullanilir.
- DNS canli karti gereksiz adapter taramasini azaltmak icin kisa sure cache kullanir.
- Tanilama paketi artik `runtime.json` ve `diagnostics.md` icinde bellek, GC, handle ve thread metrikleri verir.
- `events.jsonl` yazimi korunur, fakat ozet dosyasi her bilgi olayinda yeniden yazilmaz.
- Tanilama ZIP'i beklenmeyen log klasoru dosyalarini pakete almaz; sadece bilinen Discorder loglari allowlist'tedir.
- Header logosu ve komut butonlari daha net, daha parlak ve daha premium gorunur.
- Surum bilgileri ve updater helper metadata `2.0.21` ile hizalandi.

## Ne Degisti?

- `MainWindow`, arka plan videosunu yalnizca pencere gorunurken ve baglanti aktif/islemdeyken oynatir.
- `MediaElement` unload davranisi `Close` moduna alindi; kaynak temizligi tray/minimize/close yollarinda tekrar calisir.
- Uzak video fallback'i varsayilan kapatildi. Sadece test/demo icin `DISCORDER_BACKGROUND_VIDEO_REMOTE_FALLBACK=1` verilirse CDN fallback denenir.
- DNS karti her snapshot'ta ag adapterlerini tekrar taramak yerine 15 saniyelik canli cache kullanir.
- `DiscorderDiagnostics`, health raporuna runtime metriklerini ekler ve tanilama ZIP'ine `runtime.json` dosyasi koyar.
- Tanilama ozeti bilgi seviyesi eventlerde debounce edilir; warning/error ve health/bundle yollarinda hemen guncellenir.
- Debounce edilen son bilgi durumu kisa gecikmeyle yine `diagnostics.md` dosyasina yazilir.
- Tanilama bundle olusturma, log klasorundeki beklenmeyen dosyalari disarida birakir.
- WPF tema buton glow, logo chip ve header sunumu daha okunur hale getirildi.
- Core tanilama testi yeni runtime dosyasini ve performans bolumunu dogrular.

## Neden Onemli?

Kullanici bagli degilken bile gorev yoneticisinde yuksek bellek goruyorsa bunun onemli kismi WPF/Media Foundation video pipeline kaynakli olabilir. Bu surum, baglanti yokken videoyu calistirmayarak gorunur ekran hissini korur ama gereksiz native medya maliyetini azaltir.

Tanilama tarafinda kullanici artik sadece "hata var" dosyasi degil, ayni paketin icinde calisan surecin bellek, managed heap, GC heap, handle ve thread durumunu da gorebilir. Bu, farkli PC'lerdeki performans sikayetlerini daha net ayirmaya yardim eder.

- Guncelleme mevcut cikarilmis Discorder klasorunu hedefler.
- Paket once korumali staging alanina indirilir.
- ZIP ozeti, `.sha256.txt` dosyasi ve update manifest dogrulanmadan dosyalar uygulanmaz.
- Update helper dogru `Discorder.exe` dosyasini yeniden baslatir.
- Indirme, dogrulama veya staging hatasinda mevcut calisan klasor hedeflenmez. Uygulama asamasinda sorun olursa updater geri yukleme dener; dosya kilidi, antivirus veya yetki problemi geri yuklemeyi de engellerse kullanici dogrulanmis ZIP'i tekrar cikarmalidir.

## Guvenlik ve Guven Sinirlari

- WireSock kurucusu Authenticode, yayinci, surum ve SHA-256 kontrollerinden gecer.
- `wgcf` araci SHA-256 ile dogrulanir ve boyut siniriyla indirilir.
- Uygulama guncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden gecer.
- Tanilama runtime metrikleri sayisaldir; token, sertifika, IP listesi veya gizli anahtar icermez.
- Runtime arka plan videosu normal kullanimda GitHub/CDN'den medya cekmez; release paketi yerel `Assets/background.mp4` dosyasini icerir.
- Bu build Authenticode imzali degildir; Windows SmartScreen uyarisi gorulebilir.

## Bilinen Sinirlar

- Kod imzalama sertifikasi olmadigi icin Windows yayinci adini isletim sistemi tarafindan dogrulanmis gostermeyebilir.
- GitHub erisimi olmayan aglarda guncelleme denetimi basarisiz olabilir; mevcut surum calismaya devam eder.
- Bagli degilken arka plan videosu kapali tutulur. Test veya demo icin `DISCORDER_BACKGROUND_VIDEO_ALWAYS=1` kullanilirsa video her gorunur durumda acilir.
- Yerel arka plan videosu bozuk veya eksikse uygulama statik arka planla devam eder. Runtime CDN fallback sadece `DISCORDER_BACKGROUND_VIDEO_REMOTE_FALLBACK=1` ile acilir.
- Tarayici modu kapaliyken desteklenen tarayicilar WireSock profiline eklenmez. Ek web kapsami Windows Firewall katmaniyla yonetilir; tarayici cache'i, DoH veya DNS/IP degiskenligi nedeniyle bu katman mutlak web kill-switch olarak degerlendirilmemelidir.
